Tutorial

Suppose you have your sprite-image (that of course consists of logically measured items!) called sprite.png that contains several blocks of icons.

1. Make sure that every block of icons has the icons appearing from left to right, with equal width/height, preferrably avoiding spacing. Optionally have your mouseovers and stuff on additional lines of icons below the original ones.

2. Define generic SPRITE-block: (the .sprite-function can now be used to display anything from your sprite, not just the icons) 

.sprite(@x, @y)
{
  background-image: url(sprite.png);
  background-repeat: no-repeat;
  background-position: @x @y;
}

3. Define generic ICON-block:

.icon(@width, @height, @x: 0px, @y: 0px)
{
  .sprite(@x, @y);
  display: block;
  width: @width;
  height: @height;
}

4. For every icon-type, define a HELPER-block (my-icons are 50px x 100px and the set of my-icons is positioned at 200px x 400px in sprite.png):

.icon-my-icon-helper(@pos, @shift: 0)
{
  .icon(50px, 100px, -1 * (400px + 100px * @shift), -1 * (200px + 50px * @pos));
}

5. Define the actual ICON-block, specifying the position for every named icon, and optionally specifying your hovers:

.icon-my-icon
{
  .icon-my-icon-helper(0);
  &.first
  {
    .icon-my-icon-helper(0);
    &:hover
    {
      .icon-my-icon-helper(0, 1);
    }
  }
  &.second
  {
    .icon-my-icon-helper(1);
    &:hover
    {
      .icon-my-icon-helper(1, 1);
    }
  }
}

(You would wish for a cleaner specification of all the hovers at once, but because of most browsers lacking the background-position-x and background-position-y feature, that seems impossible, ideas welcome though!!!).

6. Now, for whatever icon you want, just write some HTML like this: [a class="my-icon first"]My First Icon[/a], you're all set!
